
# 1 说明
项目中主要用到了 **jpmml-evaluator-spark**，更详细的使用可以查看 [jpmml/jpmml-evaluator-spark](https://github.com/jpmml/jpmml-evaluator-spark)，


# 2 运行
勾选上 
* [x] intellij

# 3 打包
- [ ] intellij

使用 Idea 开发工具的 Maven 打包

使用 `mvn` 命令打包
```bash
mvn clean package

```


# 4 提交运行

提交命令如下，传入三个参数，分别是要计算的 pmml 文件路径、数据文件路径、结果输出路径。集群提交请改为 **cluster** 模式
```bash
# Client 模式，生成环境请改为 cluster 模式
#    args[0] 文件的path
#    args[1] 需要包含的列名
#    args[2] 结果的保存路径
spark-submit --master yarn --deploy-mode client \
--executor-memory 1G --num-executors 2 --executor-cores 2 \
--driver-memory 1g --total-executor-cores 2 \
--class yore.SparkWithPMML jpmml-evaluator-runtime-0.0.1-SNAPSHOT.jar \
hdfs:/home/spark/data/test.pmml hdfs:///home/spark/data/test.csv hdfs:/tmp/spark_t1

```

**原因**：出现这种错误是因为 guava 与 Spark 集群中的有版本冲突，导致的调用方法时 NoSuchMethodError。

**解决**： 在 pom.xml 的 **maven-shade-plugin** 插件下的 `<relocations>` 中添加下面代码。
它的意思就是将 `com.google` 下的所有代码重新命名打包在`com.shade.google`下，可以通过查看添加下面配置前后的的jar包。
```xml
<relocation>
    <pattern>com.google</pattern>
    <shadedPattern>com.shade.google</shadedPattern>
</relocation>

```
